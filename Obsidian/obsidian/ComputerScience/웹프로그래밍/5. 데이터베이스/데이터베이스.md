## MYSQL

MySQL을 사용하여 데이터베이스를 조작하는 데에는 여러 가지 명령어가 있습니다. 각각의 명령어를 간단히 설명하고 주의해야 할 점을 알려드리겠습니다.

1. **CREATE**: 데이터베이스, 테이블, 뷰, 인덱스 등을 생성하는 데 사용됩니다.

   ```sql
   CREATE DATABASE my_database; -- 데이터베이스 생성
   CREATE TABLE my_table (id INT, name VARCHAR(50)); -- 테이블 생성
   ```

   주의사항: 존재하는 이름의 데이터베이스나 테이블을 생성하려고 하면 오류가 발생할 수 있습니다.

2. **DROP**: 데이터베이스, 테이블, 뷰 등을 삭제합니다.

   ```sql
   DROP DATABASE my_database; -- 데이터베이스 삭제
   DROP TABLE my_table; -- 테이블 삭제
   ```

   주의사항: DROP 명령은 데이터를 완전히 삭제하므로 사용 전에 신중하게 검토해야 합니다.

3. **INSERT**: 테이블에 새로운 레코드를 삽입합니다.

   ```sql
   INSERT INTO my_table (id, name) VALUES (1, 'John'); -- 레코드 삽입
   ```

   주의사항: 삽입하려는 값의 형식이 테이블의 컬럼과 일치해야 합니다.

4. **SELECT**: 데이터를 조회합니다.

   ```sql
   SELECT * FROM my_table; -- 모든 레코드 조회
   SELECT name FROM my_table WHERE id = 1; -- 조건에 맞는 레코드 조회
   ```

   주의사항: SELECT 명령을 사용할 때 데이터베이스에 대한 읽기 권한이 필요합니다.

5. **UPDATE**: 테이블의 레코드를 수정합니다.

   ```sql
   UPDATE my_table SET name = 'Jane' WHERE id = 1; -- 레코드 수정
   ```

   주의사항: UPDATE 명령을 사용할 때 해당 레코드를 수정할 수 있는 권한이 필요합니다.

6. **DELETE**: 테이블에서 레코드를 삭제합니다.

   ```sql
   DELETE FROM my_table WHERE id = 1; -- 레코드 삭제
   ```

   주의사항: DELETE 명령은 데이터를 영구적으로 삭제하므로 주의해서 사용해야 합니다. 필요한 경우 WHERE 절을 사용하여 특정 레코드만 삭제할 수 있습니다.

또한, SQL 명령을 실행할 때 주의할 점은 다음과 같습니다:
- **보안**: 사용자 입력에 대한 SQL Injection을 방지하기 위해 항상 입력 값을 확인하고, 파라미터화된 쿼리 또는 준비된 문을 사용하세요.
- **성능**: 쿼리의 성능을 최적화하기 위해 적절한 인덱스를 생성하고, 복잡한 쿼리를 최소화하세요.
- **트랜잭션 관리**: 여러 개의 쿼리를 묶어서 원자적으로 실행되도록 트랜잭션을 관리하세요.

## 우리가 이번 학기에 자바 소스 3종류

(1) **컨트롤러 Mycontroller.java**
	new - class 했을 때 자동으로 뜸, 소스이름과 클래스 이름 똑같음
	
```java
@Controller /*@로 시작하는 걸 어노테이션, 첫글자 대문자 특징*/
public class MyController { 

	@GetMapping("/home") /*컨트롤러 메소드는 public String 으로 시작*/
	public String home(){
		return "home";
	}
}
```

(2) 엔티티 소스 
	데이터베이스 테이블 하나당 엔티티 하나
	우리가 member 테이블 만들었으므로 member.java 있어야 합니다.
```java
@Entity

public class member {
	@Id
	public String id;
	public String pw;
	public String name;
	public String phone;
	public Integer mileage;
	@CreationTimestamp
	public LocalDateTime rdate;
}
```
	멤버변수(필드)만 있는데 member 테이블 안에 6컬럼이니깐 멤버변수도 6개
	모두 public 붙여주고 자바니깐 마지막에 세미콜론 잊지말기
	
	varchar -> String, int -> Integer, datetime -> LocalDateTime 기억
	
	@어노테이션은 첫 컬럼(기본키)에 @Id
	
	컬럼 중에 LocalDateTime이 있다면 @CreatinoTimestamp 어놑테이션
